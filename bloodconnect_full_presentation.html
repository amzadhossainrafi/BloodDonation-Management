typedef struct HistoryNode {
  Donor donor;

HistoryNode* head = NULL;

/* Insert at front — O(1) */
void addHistory(Donor d) {
  HistoryNode* n = malloc(sizeof(*n));
  n->next = head;

Why Linked List here?

History grows every time someone donates. An array would get full. A linked list never gets full — it grows automatically by creating new nodes in memory.

Stack

Undo Operations — Last In, First Out (LIFO)

Code

typedef struct StackNode {
  char operation[150];
  struct StackNode* next;
} StackNode;

StackNode* top = NULL;

/* Push — O(1) */
void push(char* msg) {
  StackNode* n = malloc(sizeof(*n));
  strcpy(n->operation, msg);
  n->next = top; top = n;
}
/* Pop — O(1) */
void pop() {
  StackNode* t = top;
  top = top->next; free(t);
}

Circular Queue

Waiting List — First Come, First Served (FIFO)

Code

#define QUEUE_SIZE 10
Donor waitQueue[QUEUE_SIZE];
int front=-1, rear=-1;

void enqueue(Donor d) { /* O(1) */
  if(isFull()) return;
  if(front==-1) front=rear=0;
  else rear=(rear+1)%QUEUE_SIZE;
  waitQueue[rear]=d;
}
void dequeue() { /* O(1) */
  if(front==rear) {front=rear=-1;return;}
  front=(front+1)%QUEUE_SIZE;
}

After Donor A is served → front moves to Donor B. At index 9, rear wraps back to index 0.

Complexity: Enqueue O(1)Dequeue O(1)

Why Circular?

A normal queue wastes the empty slots at the front after dequeue. A circular queue reuses those slots — rear = (rear + 1) % SIZE — so no space is wasted.

Binary Search Tree (BST)

Fast Donor Search by ID — O(log n) average

Node & Search

typedef struct BSTNode {
  int donorIndex;
  struct BSTNode *left, *right;
} BSTNode;

/* Search — O(log n) avg */
int bstSearch(BSTNode* root, int id) {
  if(!root) return -1;
  if(id == donors[root->donorIndex].id)
    return root->donorIndex;
  if(id < donors[root->donorIndex].id)
    return bstSearch(root->left, id);
  return bstSearch(root->right, id);
}

BST Shape (IDs as keys)

     \    /    \

Inorder = 2001→2002→2003→2004→2005→2006 (sorted!)

Complexity: Search O(log n)Insert O(log n)Worst O(n)

"Sir, the fifth data structure is Binary Search Tree. I used it to search donors by ID very quickly."
"In a BST, every node has a left side and a right side. If the ID I am looking for is smaller, I go left. If it is bigger, I go right. This way, I reach the answer very fast."

Min-Heap (Priority Queue)

Emergency requests — most urgent blood type served first

Heap Operations

/* Insert — O(log n) */
void heapInsert(Donor d) {
  heap[heapSize++] = d;
  heapifyUp(heapSize-1);
}

/* Serve most urgent — O(log n) */
Donor heapExtractMin() {
  Donor top = heap[0];
  heap[0] = heap[--heapSize];
  heapifyDown(0);
  return top;
}

/* Key rule: parent always smaller */
/* parent index = (i-1) / 2 */

"Sir, the sixth data structure is Min-Heap. I used it for emergency blood requests."
"Not all blood requests are equal. O-negative blood can be given to anyone — so it is the most urgent, priority 1. AB-positive is the least urgent."

Sorting Algorithms

Bubble Sort by Name · Selection Sort by Blood Priority

Bubble Sort — sort names A to Z

void bubbleSortByName() {
  for(int i=0; i<n-1; i++)
    for(int j=0; j<n-i-1; j++)
      if(strcmp(d[j].name,d[j+1].name)>0)
        swap(d[j], d[j+1]);
}

Before: [Zara, Amzad, Rafi, John]
After pass 1: [Amzad, Rafi, John, Zara]
Final ✓: [Amzad, John, Rafi, Zara]

Selection Sort — sort by blood priority

void selectionSortByBlood() {
  for(int i=0; i<n-1; i++) {
    int mi=i;
    for(int j=i+1; j<n; j++)
      if(d[j].priority < d[mi].priority) mi=j;
    swap(d[i], d[mi]);
  }
}

Key difference: Bubble sort is stable. Selection sort makes fewer swaps — only O(n) total swaps.
